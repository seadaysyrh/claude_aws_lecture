# WAF / KMS

---

## WAF (Web Application Firewall)

WebアプリへのHTTP/HTTPSトラフィックを監視・フィルタリングするファイアウォール。

### できること

- SQLインジェクション、XSS（クロスサイトスクリプティング）のブロック
- 特定IPアドレスからのアクセス制限
- 特定の国からのアクセスを制限（地理的制限）
- レートベースのルール（同一IPからの大量リクエストをブロック）
- ボット対策

### アタッチ先

| アタッチ先 | 用途 |
|---|---|
| **CloudFront** | エッジでブロック（最も早い段階） |
| **ALB** | アプリ手前でブロック |
| **API Gateway** | APIへの不正アクセスをブロック |

### マネージドルールグループ

AWSやAWSマーケットプレイスが提供する既製のルールセット。

- `AWSManagedRulesCommonRuleSet`: 一般的な脅威への対策
- `AWSManagedRulesSQLiRuleSet`: SQLインジェクション対策
- `AWSManagedRulesAmazonIpReputationList`: 悪評高いIPのブロック

自分でルールを1から書かなくてもこれらを有効にするだけで基本的な防御ができる。

## KMS (Key Management Service)

暗号化キーの作成・管理・利用を担うサービス。

### なぜKMSが必要か

AWSの多くのサービス（S3、EBS、RDSなど）はデータを暗号化できるが、その際に使う暗号化キーをKMSで一元管理する。

### キーの種類

| 種類 | 説明 |
|---|---|
| **AWS管理キー** | AWSが自動作成・管理。ユーザーが操作不要 |
| **カスタマー管理キー (CMK)** | ユーザーが作成・管理。ローテーションやポリシー設定が可能 |
| **カスタムキーストア** | AWS CloudHSMを使って物理デバイスで管理（高セキュリティ要件向け） |

### 主な機能

- **暗号化/復号**: 最大4KBのデータを直接暗号化。大きいデータはエンベロープ暗号化
- **キーローテーション**: 年1回の自動ローテーション（CMKのみ）
- **キーポリシー**: 誰がキーを使えるかをIAMで制御
- **CloudTrail連携**: キーの利用履歴を監査ログとして記録

### エンベロープ暗号化

大きなデータを暗号化する際の仕組み。

```
1. KMSからデータキー（DEK）を生成
2. DEKでデータを暗号化
3. KMSのCMKでDEKを暗号化して一緒に保存
4. 復号時はCMKでDEKを復号 → DEKでデータを復号
```

S3のSSE-KMS、EBSの暗号化などは内部でこの方式を使っている。

## セキュリティ系サービスの全体像

```
外からの攻撃防御: WAF, Shield（DDoS対策）
通信の暗号化:    ACM（SSL証明書）
データの暗号化:  KMS
権限管理:        IAM
ログ・監査:      CloudTrail, CloudWatch
脅威検知:        GuardDuty, Security Hub
```
