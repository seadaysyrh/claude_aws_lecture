# ElastiCache

## 概要

マネージドなインメモリキャッシュサービス。RedisまたはMemcachedをAWS上でフルマネージドで使える。

DBへのクエリ結果などをメモリにキャッシュして、レスポンスを高速化しDBの負荷を下げる用途が主な使い方。

---

## Redis vs Memcached

| | Redis | Memcached |
|---|---|---|
| データ構造 | 豊富（文字列・リスト・セット・ハッシュ等） | 文字列のみ |
| 永続化 | あり（スナップショット/AOF） | なし |
| レプリケーション | あり（Multi-AZ対応） | なし |
| クラスタリング | あり | あり（シンプルな水平分散） |
| Pub/Sub | あり | なし |
| 推奨度 | **ほとんどの場合こちら** | シンプルなキャッシュだけでいいとき |

---

## Redisの主なデータ構造

| 型 | 用途例 |
|---|---|
| **String** | セッション、カウンター |
| **List** | キュー、最新N件 |
| **Set** | タグ、ユニーク管理 |
| **Sorted Set** | ランキング |
| **Hash** | ユーザープロフィール |

---

## よくあるアーキテクチャ

```
アプリ → キャッシュHIT → ElastiCache から返す（高速）
           ↓ MISS
        RDS/DynamoDB から取得 → ElastiCache に保存 → アプリに返す
```

### キャッシュ戦略

| 戦略 | 説明 |
|---|---|
| **Lazy Loading** | 読み取り時にMISSだったらDBから取得してキャッシュ |
| **Write Through** | DBへの書き込みと同時にキャッシュも更新 |
| **TTL設定** | 一定時間後にキャッシュを自動削除（データ鮮度の保持） |

## セキュリティ

- VPC内のプライベートサブネットに配置
- 保存時・転送時の暗号化
- Redis AUTH（パスワード認証）

## よく使うユースケース

- Webアプリのセッション管理
- DBクエリ結果のキャッシュ（レスポンス高速化）
- リアルタイムランキング（Sorted Set）
- レートリミット（APIの呼び出し制限）
- チャット・通知のPub/Sub
