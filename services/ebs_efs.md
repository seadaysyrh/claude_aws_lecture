# EBS / EFS

---

## EBS (Elastic Block Store)

EC2インスタンスにアタッチするブロックストレージ。PCに外付けHDDを接続するイメージ。

### 特徴

- 1つのEBSは基本的に**1つのEC2にしかアタッチできない**（io1/io2はマルチアタッチ可能）
- EC2を停止してもデータは保持される（永続化）
- 同じAZ内でしか使えない

### ボリュームタイプ

| タイプ | 種別 | 特徴 | 用途 |
|---|---|---|---|
| **gp3** | SSD | 汎用。コスパ良し（推奨） | OS、一般アプリ |
| **gp2** | SSD | 旧世代の汎用SSD | （gp3推奨） |
| **io2/io1** | SSD | 高IOPS・低レイテンシ | 高負荷DB |
| **st1** | HDD | スループット最適化 | ビッグデータ、ログ |
| **sc1** | HDD | コールドHDD。最安値 | アクセスが少ないデータ |

### スナップショット

EBSをS3に差分バックアップする機能。

```
EBS → スナップショット（S3に保存）→ 別リージョンにコピーしてDR対策も可能
```

- スナップショットからEBSを復元できる
- AMI作成時にも使われる

## EFS (Elastic File System)

**複数のEC2インスタンスで共有できる**NFSファイルシステム。

### EBSとの違い

| | EBS | EFS |
|---|---|---|
| 接続数 | 基本1台のみ | 複数EC2から同時接続 |
| アベイラビリティゾーン | 1AZに固定 | 複数AZにまたがれる |
| スケーリング | 手動でサイズ変更 | 自動でスケール |
| 用途 | OSディスク、DB | 共有ファイル置き場 |
| コスト | 安い | EBSより高い |

### ストレージクラス

| クラス | 特徴 |
|---|---|
| **EFS Standard** | 高可用性、複数AZ |
| **EFS IA (Infrequent Access)** | アクセス頻度の低いファイルを安く保存 |

### よく使うユースケース

- 複数のWebサーバーで共有するコンテンツ（画像・設定ファイル等）
- コンテナ（ECS/EKS）の共有ストレージ
- CMS（WordPressなど）のメディアファイル

---

## まとめ: ストレージの選び方

```
単体EC2のOSやDBに使いたい → EBS (gp3)
複数EC2で共有したい       → EFS
ファイルをただ保存したい  → S3
```
