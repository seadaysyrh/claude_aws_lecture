# Lambda

## 概要

サーバーレスの関数実行サービス。コードだけ書けばAWSがサーバーの管理・スケーリングを全部やってくれる。

**イベント駆動型**: 何かのトリガーがあったときだけ関数が実行され、その分だけ課金される。

---

## 基本的な動き

```
[トリガー] → [Lambda関数が起動] → [処理実行] → [終了]
```

実行時間は最大**15分**まで。それ以上かかる処理はEC2やECSを使う。

---

## トリガーの例

| トリガー | ユースケース |
|---|---|
| **API Gateway** | REST APIのバックエンド |
| **S3** | ファイルアップロード時に処理 |
| **DynamoDB Streams** | DB更新時にリアクション |
| **CloudWatch Events** | 定期実行（cronみたいに使える） |
| **SQS** | キューにメッセージが来たら処理 |
| **SNS** | 通知受信時に処理 |

---

## サポート言語

- Python
- Node.js
- Java
- Go
- Ruby
- C# (.NET)
- カスタムランタイム（コンテナイメージも使える）

---

## 課金

- **リクエスト数**: 100万リクエストまで無料（その後$0.20/100万回）
- **実行時間**: GB-秒単位。メモリ×実行時間で計算
- **無料枠**: 毎月100万リクエスト＋40万GB-秒は永続的に無料

---

## コールドスタート問題

Lambdaは一定時間使われないとコンテナが破棄される。次の実行時に起動時間（コールドスタート）が発生する。

- JavaやC#は特に遅い（JVMの起動が重い）
- **Provisioned Concurrency**: 事前にコンテナを温めておく機能（有料）

---

## 設定のポイント

| 設定 | 内容 |
|---|---|
| **メモリ** | 128MB〜10GB。CPUもメモリに比例して増える |
| **タイムアウト** | デフォルト3秒、最大15分 |
| **同時実行数** | デフォルト1000（アカウント全体） |
| **環境変数** | 設定値や秘密情報を渡す（機密はSecrets Manager推奨） |
| **レイヤー** | 共通ライブラリを複数関数で使い回す仕組み |

---

## よく使うユースケース

- API Gateway と組み合わせたREST API
- S3にファイルを置いたら自動でサムネイル生成
- 定期バッチ処理（夜間集計など）
- Webhookの受け口
