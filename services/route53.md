# Route 53

## 概要

AWSのDNSサービス。ドメイン名をIPアドレスに変換する仕組みを提供する。ドメイン登録・購入もできる。

名前の由来はDNSが使うポート番号「53番」から。

---

## 主な機能

| 機能 | 説明 |
|---|---|
| **ドメイン登録** | AWSでドメインを購入・管理できる |
| **DNSホスティング** | ゾーン（ドメイン）のDNSレコードを管理 |
| **ヘルスチェック** | エンドポイントの死活監視 |
| **トラフィックルーティング** | 複数エンドポイントへのルーティングポリシー |

---

## レコードタイプ

| レコード | 説明 |
|---|---|
| **A** | ドメイン → IPv4アドレス |
| **AAAA** | ドメイン → IPv6アドレス |
| **CNAME** | ドメイン → 別のドメイン名（エイリアス） |
| **MX** | メールサーバーの指定 |
| **TXT** | テキスト情報（ドメイン所有確認などに使う） |
| **Alias** | Route 53独自。AWSリソース（ALB、CloudFrontなど）に向けるときに使う |

> CNAMEとAliasの違い: Aliasはゾーンapexドメイン（example.comそのもの）でも使えてDNSクエリが無料。AWSリソースへの向き先はAliasを使うのが推奨。

---

## ルーティングポリシー

| ポリシー | 説明 | ユースケース |
|---|---|---|
| **シンプル** | 1つのリソースに向ける | 基本的な用途 |
| **加重（Weighted）** | 重み付けで複数リソースに分散 | カナリアデプロイ（新バージョンに10%だけ流すなど） |
| **レイテンシー** | レイテンシーが最も低いリージョンに向ける | マルチリージョン構成 |
| **フェイルオーバー** | プライマリ障害時にセカンダリに切り替え | DR構成 |
| **地理的（Geolocation）** | ユーザーの場所に応じてルーティング | 地域ごとにコンテンツを分ける |
| **複数値回答** | 複数IPを返してクライアント側で選ばせる | シンプルなロードバランシング |

---

## ヘルスチェック

エンドポイントが正常かどうかを定期的に確認。

- HTTP/HTTPS/TCPチェックに対応
- フェイルオーバールーティングと組み合わせて自動切り替え
- CloudWatchと連携してアラート通知も可能

---

## よく使う構成

```
example.com → Route 53 → ALB → EC2/ECS
                       → CloudFront（静的コンテンツ）
```

```
マルチリージョンフェイルオーバー:
Route 53 → プライマリリージョン（東京）
         ↓ 障害時
         → セカンダリリージョン（大阪）
```
