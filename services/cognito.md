# Cognito

## 概要

アプリケーション向けのユーザー認証・認可サービス。ユーザーのサインアップ、ログイン、トークン発行などの機能をフルマネージドで提供する。

自前で認証機能を実装する手間を省ける。

---

## 2つのコンポーネント

### ユーザープール (User Pool)

ユーザーディレクトリ（ユーザー情報の管理）とID/パスワード認証を提供する。

- ユーザーの登録・管理
- メール/SMS検証
- パスワードポリシー設定
- MFA対応
- JWT（IDトークン・アクセストークン）の発行

### IDプール (Identity Pool / Federated Identities)

認証済みユーザーにAWSリソースへの一時的なアクセス権限を付与する。

- ユーザープールや外部IdPで認証したユーザーにIAMロールを割り当て
- 例: 認証済みユーザーだけS3の特定バケットにアクセスできるようにする

---

## ソーシャルログイン連携

ユーザープールに外部のIDプロバイダー（IdP）を連携できる。

| プロバイダー | 説明 |
|---|---|
| Google | Googleアカウントでログイン |
| Facebook | Facebookアカウントでログイン |
| Apple | Appleアカウントでログイン |
| SAML 2.0 | 企業のActive Directoryなど |
| OIDC | OpenID Connect対応のIdP |

---

## 典型的な使い方

### SPAとAPIの認証

```
ユーザー
  ↓ ログイン
Cognito ユーザープール → JWTトークン発行
  ↓ トークンを付けてリクエスト
API Gateway → トークン検証 → Lambda → DynamoDB
```

### モバイルアプリ + S3

```
ユーザー認証（ユーザープール）
  ↓
IDプール → 一時IAM認証情報
  ↓
S3の自分専用フォルダに直接アクセス
```

---

## Amplifyとの連携

AWS Amplifyを使うと、Cognitoの認証UIコンポーネントをフロントエンドに簡単に組み込める。

```javascript
import { Authenticator } from '@aws-amplify/ui-react';
// これだけでログイン画面が生成される
```

## よく使うユースケース

- WebアプリやモバイルアプリのユーザーログインID/PW管理
- ソーシャルログイン（Google/Apple等）の実装
- 認証済みユーザーのみS3やAPIにアクセスさせる
- B2Cサービスの会員管理
