# CloudFront

## 概要

AWSのCDN（Content Delivery Network）サービス。世界中にあるエッジロケーション（キャッシュサーバー）からコンテンツを配信することで、ユーザーへのレスポンスを高速化する。

---

## 仕組み

```
ユーザー（東京）
    ↓ 最寄りのエッジロケーションにアクセス
エッジロケーション（東京）
    ├─ キャッシュHIT → そのまま返す（速い）
    └─ キャッシュMISS → オリジンから取得してキャッシュ
                          ↓
                        オリジン（S3 / ALB / EC2 など）
```

エッジロケーションは世界450+拠点。

---

## オリジン

CloudFrontが実際のコンテンツを取りに行く場所。

| オリジン | 用途 |
|---|---|
| **S3バケット** | 静的コンテンツ配信 |
| **ALB / EC2** | 動的コンテンツ配信 |
| **API Gateway** | API高速化 |
| **カスタムオリジン** | AWS外のHTTPサーバー |

複数オリジンをビヘイビア（パスパターン）で使い分けることも可能。
```
/api/*   → ALB（動的）
/*       → S3（静的）
```

---

## 主要機能

### キャッシュ制御

- **TTL**: キャッシュの有効期限（Cache-Controlヘッダーで制御）
- **キャッシュの無効化**: 特定パスのキャッシュを強制クリア（デプロイ後などに使う）

### HTTPS対応

- **ACM（AWS Certificate Manager）** で無料のSSL証明書を発行してHTTPS化
- HTTP→HTTPSの自動リダイレクトも設定可能

### OAC (Origin Access Control)

S3をオリジンにする場合、CloudFront経由でしかアクセスできないよう制限する仕組み。S3への直接アクセスをブロックしてセキュリティ強化。

### WAFとの統合

CloudFrontにWAFを付けて、エッジでSQLインジェクションやXSSをブロックできる。

### Lambda@Edge / CloudFront Functions

エッジロケーションで軽量な処理を実行する仕組み。

```
用途例:
- リクエストのリダイレクト
- 認証ヘッダーの追加
- URLの書き換え
- A/Bテストのルーティング
```

---

## 料金

- **データ転送量**: リージョンやエッジロケーションによって異なる
- **リクエスト数**: HTTPSリクエストは若干高い
- **無効化リクエスト**: 月1000パスまで無料

S3への直接配信よりCloudFront経由の方がコストが安くなるケースも多い。

---

## よく使う構成

```
静的Webサイト:
Route 53 → CloudFront → S3（静的ファイル）

SPA + API:
Route 53 → CloudFront → /        → S3（React/Vue build）
                      → /api/*   → ALB → EC2/Lambda
```
